# L2.1 Validation Fixes — Historical

> **Status:** Historical snapshot. Defects described here were resolved
> during the L2.1 cycle. The current state is in `docs/architecture.md`
> and `docs/pitfalls/`.

## What was fixed

Three validation defects were discovered during the L2.1 cycle:

### 1. Default path unification

`services` were reading the dataset path from different sources. Unified
into `config.py` as the single source of truth. See
`docs/pitfalls/default-path-drift.md`.

### 2. `schema_version` enforcement

v2 entities now require `schema_version: 2`. See
`docs/pitfalls/validation-strictness.md`.

### 3. Relation type validation

Unknown relation types are now errors. The valid list is enforced by
`cmdb/rules/relations.py:VALID_RELATION_TYPES`. See
`docs/pitfalls/relation-type-sync.md`.

## Why this matters

Before L2.1, validation passed when it should have failed. The L2.1
strict-mode rules make invalid states unrepresentable in the dataset.
